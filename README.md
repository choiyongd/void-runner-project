# VOID RUNNER

Three.js 기반 3D 우주 탐사/러너 게임. 순수 HTML/CSS/JS 단일 파일(`index.html`)로 작성되어 있으며, Three.js는 cdnjs CDN에서 로드합니다. 별도 빌드 과정이나 패키지 설치 없이 브라우저에서 바로 실행됩니다.

## 실행 방법
`index.html`을 브라우저로 열면 바로 플레이할 수 있습니다. (로컬 서버 없이도 동작하지만, 일부 브라우저 보안 정책 때문에 `python3 -m http.server` 같은 간단한 로컬 서버로 열기를 권장합니다.)

## 주요 시스템
- 3D 비행/조작 (WASD·방향키 또는 터치 스틱, 부스트)
- 소행성 충돌 물리(내구도) + 파괴 시 자원(스크랩) 드랍
- 인벤토리 시스템 (고철 → 정제 합금 변환, 데이터 조각)
- 보상형 광고 시스템 (하루 3회 제한, localStorage 기반)
- 함선 10종 교체 (실제 지구 발사체 6종 + 오리지널 창작 함선 4종)
- 함선 잔해/폐허 정거장 청소 시스템
- 실제 우주 구조를 반영한 6단계 구역(태양계 → 항성 이웃 → 은하 나선팔 → 은하 중심 → 국부은하군 → 초은하단) + 항성 근접 정보 팝업
- 식민화 시스템 (누적 가능, 각 1회용 리스폰)
- 웜홀을 통한 구역 이동
- 외계 통신 시스템 (누적 교신 10회마다 다양한 형태의 외계 조우 이벤트)
- 퀘이사·혜성 등 배경 천체
- 파티클 기반 스러스터 화염/트레일, 충돌 스파크 이펙트

## Supabase 연동 (글로벌 리더보드 + 클라우드 세이브)
`index.html`에 Supabase 프로젝트가 연결되어 있습니다. 최초 1회 아래 설정이 필요합니다.

1. Supabase Dashboard → SQL Editor에서 [`supabase/schema.sql`](supabase/schema.sql) 내용을 실행해 `leaderboard`, `cloud_saves` 테이블과 RLS 정책을 생성합니다.
2. Dashboard → Authentication → Sign In / Providers에서 **Anonymous Sign-Ins**를 활성화합니다. (클라우드 세이브는 기기별 익명 계정으로 동작하며, 이 설정이 꺼져 있으면 리더보드는 정상 동작하지만 클라우드 세이브는 콘솔 경고와 함께 조용히 비활성화됩니다.)

연동 후:
- 게임오버 화면에서 이름을 입력하고 "기록 등록"을 누르면 `leaderboard`에 점수가 저장되고, 시작 화면과 게임오버 화면에 TOP 10이 표시됩니다.
- 최고점수·식민지·함선 선택·자원(고철/합금/데이터 조각)은 `cloud_saves`에 자동 저장되어 같은 브라우저에서 재접속해도 유지됩니다.
